## POST /api/auth/login

Authenticates a user with email and password credentials and returns an access token.

<Note>
  This endpoint does not require authentication. Use the returned access token for subsequent authenticated requests.
</Note>

### Request Body

<ParamField path="email" type="string" required>
  User's registered email address
</ParamField>

<ParamField path="password" type="string" required>
  User's password
</ParamField>

### Response

<ResponseField name="access_token" type="string">
  JWT access token to be used for authentication in subsequent requests
</ResponseField>

<ResponseField name="token_type" type="string">
  Token type, always "bearer"
</ResponseField>

### Request Example

<CodeGroup>
```bash cURL
curl -X POST https://api.taskpro.com/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "john.doe@example.com",
    "password": "SecurePass123!"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.taskpro.com/api/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email: 'john.doe@example.com',
    password: 'SecurePass123!'
  })
});

const data = await response.json();
const token = data.access_token;

// Use token in subsequent requests
const authenticatedResponse = await fetch('https://api.taskpro.com/api/auth/me', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});
```

```python Python
import requests

response = requests.post(
    'https://api.taskpro.com/api/auth/login',
    json={
        'email': 'john.doe@example.com',
        'password': 'SecurePass123!'
    }
)

data = response.json()
token = data['access_token']

# Use token in subsequent requests
headers = {'Authorization': f'Bearer {token}'}
auth_response = requests.get(
    'https://api.taskpro.com/api/auth/me',
    headers=headers
)
```
</CodeGroup>

### Status Codes

| Status Code | Description |
|-------------|-------------|
| 200 | Authentication successful, token returned |
| 400 | Missing or invalid request body |
| 401 | Invalid credentials |
| 500 | Internal server error |

### Using the Access Token

<Info>
  Include the access token in the `Authorization` header of subsequent requests:
  
  `Authorization: Bearer <access_token>`
</Info>
