## Overview

Tags provide a flexible categorization system that works across tasks, folders, and subtasks. Unlike folders which provide hierarchical organization, tags allow you to create cross-cutting categories that can be applied to any resource.

## Tag Model

The Tag entity structure:

```kotlin
@Entity
@Table(name = "tags")
data class Tag(
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    val tagId: Long? = null,
    
    val userId: Long,
    val tagName: String,
    val color: String? = null,
    val createdAt: LocalDateTime? = null,
    val updatedAt: LocalDateTime? = null,
    
    var user: User? = null,
    var tasks: Set<Task> = emptySet(),
    var subtasks: Set<Subtask> = emptySet(),
    var folders: Set<Folder> = emptySet()
)
```

## Tag Properties

| Property | Type | Description |
|----------|------|-------------|
| `tagId` | Long | Unique identifier (auto-generated) |
| `userId` | Long | ID of the user who owns the tag |
| `tagName` | String | Tag name (required, max 100 chars) |
| `color` | String | Hex color code for visual identification (max 7 chars) |
| `createdAt` | LocalDateTime | Tag creation timestamp |
| `updatedAt` | LocalDateTime | Last modification timestamp |

### Relationships

| Property | Type | Description |
|----------|------|-------------|
| `user` | User | Owner of the tag |
| `tasks` | Set&lt;Task&gt; | Tasks that have this tag |
| `subtasks` | Set&lt;Subtask&gt; | Subtasks that have this tag |
| `folders` | Set&lt;Folder&gt; | Folders that have this tag |

## Creating a Tag

Create a new tag with a name and optional color:

```bash
curl -X POST https://api.taskpro.com/api/tags \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -d '{
    "tagName": "urgent",
    "color": "#EF4444"
  }'
```

### Request DTO

```kotlin
data class TagCreateDto(
    val tagName: String,
    val color: String? = null
)
```

### Response

```json
{
  "tagId": 1,
  "userId": 1,
  "tagName": "urgent",
  "color": "#EF4444",
  "createdAt": "2024-03-15T14:30:00",
  "updatedAt": "2024-03-15T14:30:00"
}
```

## Retrieving Tags

### Get All Tags

Retrieve all tags for the authenticated user:

```bash
curl -X GET https://api.taskpro.com/api/tags \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

### Get Single Tag

Retrieve a specific tag by ID:

```bash
curl -X GET https://api.taskpro.com/api/tags/1 \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

## Updating a Tag

Update tag name or color:

```bash
curl -X PUT https://api.taskpro.com/api/tags/1 \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -d '{
    "tagName": "high-priority",
    "color": "#DC2626"
  }'
```

### Update DTO

```kotlin
data class TagUpdateDto(
    val tagName: String? = null,
    val color: String? = null
)
```

<Note>
  When you update a tag, the changes are reflected across all tasks, folders, and subtasks that use it.
</Note>

## Deleting a Tag

Permanently delete a tag:

```bash
curl -X DELETE https://api.taskpro.com/api/tags/1 \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

<Warning>
  Deleting a tag removes it from all associated tasks, folders, and subtasks. The resources themselves are not deleted, only the tag association.
</Warning>

## Applying Tags to Resources

### Tags on Tasks

Apply tags when creating or updating tasks:

```bash
curl -X POST https://api.taskpro.com/api/tasks \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -d '{
    "title": "Fix authentication bug",
    "tagIds": [1, 3],
    "priority": 2
  }'
```

The response includes the full tag objects:

```json
{
  "taskId": 42,
  "title": "Fix authentication bug",
  "tags": [
    {
      "tagId": 1,
      "tagName": "urgent",
      "color": "#EF4444"
    },
    {
      "tagId": 3,
      "tagName": "backend",
      "color": "#10B981"
    }
  ],
  ...
}
```

### Tags on Folders

Apply tags to folders for categorization:

```bash
curl -X POST https://api.taskpro.com/api/folders \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -d '{
    "name": "Q1 Projects",
    "tagIds": [2, 5],
    "color": "#3B82F6"
  }'
```

### Tags on Subtasks

Subtasks also support tags for fine-grained categorization:

```bash
curl -X POST https://api.taskpro.com/api/subtasks \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -d '{
    "taskId": 42,
    "title": "Write unit tests",
    "tagIds": [3, 7]
  }'
```

## Many-to-Many Relationships

Tags use many-to-many relationships with join tables:

### Task Tags

```kotlin
@ManyToMany
@JoinTable(
    name = "task_tags",
    joinColumns = [JoinColumn(name = "task_id")],
    inverseJoinColumns = [JoinColumn(name = "tag_id")]
)
var tags: Set<Tag> = emptySet()
```

Database table `task_tags`:
```sql
CREATE TABLE task_tags (
    task_id BIGINT REFERENCES tasks(task_id),
    tag_id BIGINT REFERENCES tags(tag_id),
    PRIMARY KEY (task_id, tag_id)
);
```

### Folder Tags

```kotlin
@ManyToMany
@JoinTable(
    name = "folder_tags",
    joinColumns = [JoinColumn(name = "folder_id")],
    inverseJoinColumns = [JoinColumn(name = "tag_id")]
)
var tags: Set<Tag> = emptySet()
```

### Subtask Tags

```kotlin
@ManyToMany
@JoinTable(
    name = "subtask_tags",
    joinColumns = [JoinColumn(name = "subtask_id")],
    inverseJoinColumns = [JoinColumn(name = "tag_id")]
)
var tags: Set<Tag> = emptySet()
```

## Best Practices

<Steps>
  <Step title="Keep names short">
    Use concise, lowercase names with hyphens: `high-priority` instead of `High Priority Task`.
  </Step>
  <Step title="Use consistent colors">
    Establish a color scheme and stick to it across similar tag categories.
  </Step>
  <Step title="Limit tag count">
    Don't create too many tags. Aim for 10-20 core tags that cover your main categories.
  </Step>
  <Step title="Combine with folders">
    Use folders for hierarchy (projects) and tags for cross-cutting concerns (priority, status).
  </Step>
  <Step title="Review and clean up">
    Periodically review tags and delete unused ones to keep the system manageable.
  </Step>
</Steps>

## Filtering with Tags

While tags are applied to resources, the API doesn't currently support filtering tasks by tag ID in the query parameters. To filter by tags:

1. Retrieve all tasks
2. Filter client-side based on tag IDs
3. Or implement custom filtering in your application layer

<Tip>
  Future API versions may include tag-based filtering endpoints like `GET /api/tasks?tagId=1`.
</Tip>
