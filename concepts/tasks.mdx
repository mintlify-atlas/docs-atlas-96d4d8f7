## Overview

Tasks are the core resource in Task Pro API. Each task represents a single actionable item with properties like title, description, status, priority, due date, and more. Tasks can be organized into folders, tagged for categorization, and broken down into subtasks.

## Task Model

The Task entity is defined in the database with the following structure:

```kotlin
@Entity
@Table(name = "tasks")
data class Task(
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    val taskId: Long? = null,
    
    val userId: Long,
    val folderId: Long? = null,
    val title: String,
    val description: String? = null,
    val durationTagId: Int? = null,
    val priority: Int = 0,
    val status: Status = Status.PENDING,
    val dueDate: LocalDate? = null,
    val dueTime: LocalTime? = null,
    val completedAt: LocalDateTime? = null,
    val displayOrder: Int = 0,
    val isArchived: Boolean = false,
    val createdAt: LocalDateTime? = null,
    val updatedAt: LocalDateTime? = null,
    
    var tags: Set<Tag> = emptySet(),
    var folder: Folder? = null,
    var user: User? = null,
    var subtasks: Set<Subtask> = emptySet()
)
```

## Task Properties

### Core Properties

| Property | Type | Description |
|----------|------|-------------|
| `taskId` | Long | Unique identifier (auto-generated) |
| `userId` | Long | ID of the user who owns the task |
| `title` | String | Task title (required, max 255 chars) |
| `description` | String | Detailed task description (optional, text) |

### Organization

| Property | Type | Description |
|----------|------|-------------|
| `folderId` | Long | ID of the parent folder (optional) |
| `tags` | Set&lt;Tag&gt; | Associated tags for categorization |
| `displayOrder` | Int | Order for sorting tasks (default: 0) |

### Status & Priority

| Property | Type | Description |
|----------|------|-------------|
| `status` | Enum | Task status: PENDING, IN_PROGRESS, COMPLETED |
| `priority` | Int | Priority level (higher = more important) |
| `isArchived` | Boolean | Whether the task is archived |

### Time Management

| Property | Type | Description |
|----------|------|-------------|
| `dueDate` | LocalDate | Target completion date |
| `dueTime` | LocalTime | Specific time for the due date |
| `durationTagId` | Int | Reference to duration estimation |
| `completedAt` | LocalDateTime | Timestamp when task was completed |
| `createdAt` | LocalDateTime | Task creation timestamp |
| `updatedAt` | LocalDateTime | Last modification timestamp |

### Relationships

| Property | Type | Description |
|----------|------|-------------|
| `subtasks` | Set&lt;Subtask&gt; | Child subtasks belonging to this task |
| `folder` | Folder | Parent folder (if assigned) |
| `user` | User | Owner of the task |

## Creating a Task

Create a new task with the required and optional properties:

```bash
curl -X POST https://api.taskpro.com/api/tasks \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -d '{
    "title": "Design new landing page",
    "description": "Create wireframes and mockups for the new landing page",
    "folderId": 5,
    "tagIds": [1, 3],
    "priority": 2,
    "status": "pending",
    "dueDate": "2024-03-25",
    "dueTime": "17:00:00",
    "displayOrder": 1
  }'
```

## Retrieving Tasks

### Get All Tasks

Retrieve all tasks for the authenticated user with optional filters:

```bash
curl -X GET "https://api.taskpro.com/api/tasks" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

### Filter by Folder

Get tasks in a specific folder:

```bash
curl -X GET "https://api.taskpro.com/api/tasks?folderId=5" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

### Filter by Status

Get tasks with a specific status:

```bash
curl -X GET "https://api.taskpro.com/api/tasks?status=in_progress" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

### Include Archived Tasks

By default, archived tasks are excluded. Include them with:

```bash
curl -X GET "https://api.taskpro.com/api/tasks?includeArchived=true" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

## Task Progress

When retrieving tasks, the API returns `TaskWithProgressDto` which includes subtask completion information:

```json
{
  "taskId": 42,
  "title": "Design new landing page",
  "totalSubtasks": 5,
  "completedSubtasks": 3,
  "completionPercentage": 60.0,
  ...
}
```

<Info>
  The completion percentage is automatically calculated based on the number of completed subtasks versus total subtasks.
</Info>

## Updating a Task

Update any task properties using the task ID:

```bash
curl -X PUT https://api.taskpro.com/api/tasks/42 \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -d '{
    "status": "in_progress",
    "priority": 3
  }'
```

### Update DTO

```kotlin
data class TaskUpdateDto(
    val title: String? = null,
    val description: String? = null,
    val folderId: Long? = null,
    val tagIds: List<Long>? = null,
    val durationTagId: Int? = null,
    val priority: Int? = null,
    val status: String? = null,
    val dueDate: LocalDate? = null,
    val dueTime: LocalTime? = null,
    val displayOrder: Int? = null,
    val isArchived: Boolean? = null
)
```

<Note>
  All fields in the update DTO are optional. Only include the fields you want to update.
</Note>

## Deleting a Task

Permanently delete a task:

```bash
curl -X DELETE https://api.taskpro.com/api/tasks/42 \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

<Warning>
  Deleting a task also deletes all associated subtasks due to cascade deletion. This action cannot be undone.
</Warning>

## Task Relationships

### Many-to-One: Task → User

Each task belongs to exactly one user. The `userId` is set automatically from the authenticated user's token.

### Many-to-One: Task → Folder

A task can optionally belong to a folder for organization:

```kotlin
@ManyToOne(fetch = FetchType.LAZY)
@JoinColumn(name = "folder_id", insertable = false, updatable = false)
var folder: Folder? = null
```

### Many-to-Many: Task ↔ Tag

Tasks can have multiple tags, and tags can be applied to multiple tasks:

```kotlin
@ManyToMany
@JoinTable(
    name = "task_tags",
    joinColumns = [JoinColumn(name = "task_id")],
    inverseJoinColumns = [JoinColumn(name = "tag_id")]
)
var tags: Set<Tag> = emptySet()
```

### One-to-Many: Task → Subtask

A task can have multiple subtasks for breaking down work:

```kotlin
@OneToMany(mappedBy = "task", cascade = [CascadeType.ALL], orphanRemoval = true)
var subtasks: Set<Subtask> = emptySet()
```
