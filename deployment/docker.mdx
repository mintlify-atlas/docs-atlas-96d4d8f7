## Overview

Task Pro API is containerized using Docker for easy deployment. The application uses a multi-stage build process to create a lightweight production image based on Eclipse Temurin JRE.

## Dockerfile

The application includes a production-ready Dockerfile:

```dockerfile
### Build stage
FROM gradle:9.3.1-jdk17 AS build
WORKDIR /home/gradle/project
COPY --chown=gradle:gradle . /home/gradle/project
# Ensure the wrapper is executable and run the Gradle wrapper to use the project-specific Gradle version
RUN chmod +x ./gradlew && ./gradlew bootJar --no-daemon --warning-mode=all

### Run stage
FROM eclipse-temurin:17-jre-jammy
WORKDIR /app
COPY --from=build /home/gradle/project/build/libs/task-pro-api-0.0.1-SNAPSHOT.jar app.jar

ENV JAVA_TOOL_OPTIONS="-Xms256m -Xmx512m"
EXPOSE 8080
ENTRYPORT ["java","-jar","/app/app.jar"]
```

### Build Stages

<Steps>
  <Step title="Build Stage">
    Uses Gradle 9.3.1 with JDK 17 to compile and package the application into a JAR file.
  </Step>
  <Step title="Run Stage">
    Uses lightweight Eclipse Temurin JRE 17 runtime with the compiled JAR, reducing final image size.
  </Step>
</Steps>

## Building the Docker Image

### Local Build

Build the Docker image locally:

```bash
docker build -t task-pro-api:latest .
```

<Info>
  The build process takes several minutes on first run as Gradle downloads dependencies. Subsequent builds are faster due to Docker layer caching.
</Info>

### Build with Custom Tag

Tag for your registry:

```bash
docker build -t your-registry/task-pro-api:v1.0.0 .
```

### View Built Image

```bash
docker images | grep task-pro-api
```

## Running with Docker

### Basic Run

Run the container with default settings:

```bash
docker run -d \
  --name task-pro-api \
  -p 8080:8080 \
  task-pro-api:latest
```

<Warning>
  This basic configuration uses an in-memory H2 database. Data will be lost when the container stops. For production, always configure a PostgreSQL database.
</Warning>

### Run with Environment Variables

Run with PostgreSQL connection:

```bash
docker run -d \
  --name task-pro-api \
  -p 8080:8080 \
  -e SPRING_PROFILES_ACTIVE=prod \
  -e SPRING_DATASOURCE_URL=jdbc:postgresql://db:5432/task_pro_db \
  -e SPRING_DATASOURCE_USERNAME=task_pro_user \
  -e SPRING_DATASOURCE_PASSWORD=task_pro_pass \
  -e SPRING_JPA_HBM2DDL=validate \
  -e APP_CORS_ALLOWED_ORIGINS=https://app.example.com \
  task-pro-api:latest
```

### Memory Configuration

The default memory limits are:
- Initial heap: 256MB (`-Xms256m`)
- Maximum heap: 512MB (`-Xmx512m`)

Override with custom `JAVA_TOOL_OPTIONS`:

```bash
docker run -d \
  --name task-pro-api \
  -p 8080:8080 \
  -e JAVA_TOOL_OPTIONS="-Xms512m -Xmx1024m" \
  task-pro-api:latest
```

## Docker Compose

### Development Setup

The repository includes `docker-compose.postgres.yml` for local development with PostgreSQL:

```yaml
version: '3.8'

services:
  db:
    image: postgres:15-alpine
    container_name: task_pro_postgres
    environment:
      POSTGRES_DB: task_pro_db
      POSTGRES_USER: task_pro_user
      POSTGRES_PASSWORD: task_pro_pass
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U task_pro_user"]
      interval: 10s
      timeout: 5s
      retries: 5

volumes:
  postgres_data:
```

### Start PostgreSQL

```bash
docker compose -f docker-compose.postgres.yml up -d
```

### Stop PostgreSQL

```bash
docker compose -f docker-compose.postgres.yml down
```

### Complete Stack (API + Database)

Create a `docker-compose.yml` for the full stack:

```yaml
version: '3.8'

services:
  db:
    image: postgres:15-alpine
    container_name: task_pro_postgres
    environment:
      POSTGRES_DB: task_pro_db
      POSTGRES_USER: task_pro_user
      POSTGRES_PASSWORD: ${DB_PASSWORD:-task_pro_pass}
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U task_pro_user"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - task_pro_network

  api:
    build: .
    container_name: task_pro_api
    ports:
      - "8080:8080"
    environment:
      SPRING_PROFILES_ACTIVE: prod
      SPRING_DATASOURCE_URL: jdbc:postgresql://db:5432/task_pro_db
      SPRING_DATASOURCE_USERNAME: task_pro_user
      SPRING_DATASOURCE_PASSWORD: ${DB_PASSWORD:-task_pro_pass}
      SPRING_JPA_HBM2DDL: validate
      APP_CORS_ALLOWED_ORIGINS: ${CORS_ORIGINS:-*}
    depends_on:
      db:
        condition: service_healthy
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8080/actuator/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s
    networks:
      - task_pro_network

volumes:
  postgres_data:

networks:
  task_pro_network:
    driver: bridge
```

### Start the Stack

```bash
# With default values
docker compose up -d

# With custom environment variables
DB_PASSWORD=secure_password CORS_ORIGINS=https://app.example.com docker compose up -d
```

### View Logs

```bash
# All services
docker compose logs -f

# API only
docker compose logs -f api

# Database only
docker compose logs -f db
```

### Check Service Health

```bash
docker compose ps
```

## Health Checks

The API includes Spring Boot Actuator health checks:

```bash
curl http://localhost:8080/actuator/health
```

### Response

```json
{
  "status": "UP"
}
```

### Docker Health Check

The Docker Compose configuration includes a health check:

```yaml
healthcheck:
  test: ["CMD", "curl", "-f", "http://localhost:8080/actuator/health"]
  interval: 30s
  timeout: 10s
  retries: 3
  start_period: 40s
```

<Note>
  The `start_period` gives the application 40 seconds to start before health checks begin, allowing time for database migrations and initialization.
</Note>

## Production Deployment

### Using Easypanel (Recommended)

Easypanel simplifies VPS deployment:

<Steps>
  <Step title="Build and push image">
    ```bash
    docker build -t your-repo/task-pro-api:latest .
    docker push your-repo/task-pro-api:latest
    ```
  </Step>
  <Step title="Create app in Easypanel">
    - Choose "Docker Image" type
    - Enter image: `your-repo/task-pro-api:latest`
  </Step>
  <Step title="Configure environment variables">
    Set required environment variables in the Easypanel UI (see [Environment Variables](/deployment/environment-variables)).
  </Step>
  <Step title="Add PostgreSQL service">
    - Create a PostgreSQL service in Easypanel
    - Link it to your API service
    - Update `SPRING_DATASOURCE_URL` to point to the database
  </Step>
  <Step title="Configure health check">
    - Path: `/actuator/health`
    - Port: `8080`
    - Interval: 30 seconds
  </Step>
</Steps>

### Using Docker Compose on VPS

<Steps>
  <Step title="Clone repository">
    ```bash
    git clone https://github.com/your-org/task-pro-api.git
    cd task-pro-api
    ```
  </Step>
  <Step title="Create .env file">
    ```bash
    cat > .env << EOF
    DB_PASSWORD=your_secure_password
    CORS_ORIGINS=https://your-frontend.com
    JWT_SECRET=your_jwt_secret_key
    EOF
    ```
  </Step>
  <Step title="Start services">
    ```bash
    docker compose up -d
    ```
  </Step>
  <Step title="Verify deployment">
    ```bash
    curl http://localhost:8080/actuator/health
    ```
  </Step>
</Steps>

## Resource Requirements

### Minimum

- CPU: 1 vCPU
- RAM: 512MB (JVM + overhead)
- Disk: 500MB (image + logs)

### Recommended

- CPU: 2 vCPUs
- RAM: 1GB
- Disk: 2GB

### Database (PostgreSQL)

- CPU: 1 vCPU
- RAM: 512MB
- Disk: 10GB (grows with data)

## Troubleshooting

### Container Won't Start

```bash
# Check logs
docker logs task-pro-api

# Check if port is already in use
lsof -i :8080
```

### Database Connection Issues

```bash
# Test database connectivity from API container
docker exec -it task-pro-api sh
ping db
```

### Out of Memory

Increase JVM heap size:

```bash
docker run -d \
  -e JAVA_TOOL_OPTIONS="-Xms512m -Xmx1024m" \
  task-pro-api:latest
```

### Application Not Ready

Check health endpoint:

```bash
# From host
curl http://localhost:8080/actuator/health

# From inside container
docker exec task-pro-api curl http://localhost:8080/actuator/health
```
